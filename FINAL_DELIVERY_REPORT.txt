ROLE-BASED DASHBOARD SYSTEM:

The system uses Redux state to detect user role from JWT token and renders
appropriate dashboard based on role:

  if (user.role === 'admin') → Admin Dashboard
  if (user.role === 'hr_manager') → HR Manager Dashboard
  if (user.role === 'manager') → Manager Dashboard
  if (user.role === 'employee') → Employee Dashboard

MENU FILTERING:

The navbar dynamically filters menu items based on user role:

  Admin: 7 items (all features)
  HR Manager: 6 items (no Roles)
  Manager: 4 items (limited access)
  Employee: 4 items (self-service only)

COLOR CODING:

Each role has a unique color scheme for easy identification:

  Admin: Purple gradient (#667eea → #764ba2)
  HR Manager: Dark Purple gradient (#764ba2 → #9b59b6)
  Manager: Pink gradient (#f093fb → #f5576c)
  Employee: Blue gradient (#4facfe → #00f2fe)

COMMON ISSUES & SOLUTIONS:

Issue: Application won't load
  Solution:
    1. Check both servers are running
    2. Check http://localhost:5000 (backend)
    3. Check http://localhost:3000 (frontend)
    4. Check browser console (F12) for errors

Issue: Login fails
  Solution:
    1. Verify backend is running
    2. Check credentials
    3. Check browser console (F12)
    4. Check network tab for API errors

Issue: Dashboard not showing
  Solution:
    1. Clear browser cache (Ctrl+Shift+Delete)
    2. Reload page (Ctrl+R)
    3. Logout and login again
    4. Check browser console for errors

Issue: Menu items not showing
  Solution:
    1. Verify user role is correct
    2. Check JWT token in storage
    3. Logout and login again
    4. Check Redux state in browser DevTools

Issue: Styling looks wrong
  Solution:
    1. Clear cache and reload
    2. Check browser zoom (100%)
    3. Rebuild frontend if needed
    4. Check CSS files are loaded

Issue: Responsive design not working
  Solution:
    1. Check viewport meta tag
    2. Clear browser cache
    3. Reload page
    4. Check CSS media queries
