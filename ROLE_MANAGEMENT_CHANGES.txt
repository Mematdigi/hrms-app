NEW API ENDPOINTS (3 ENDPOINTS)
================================================================================

1. GET /api/roles/users
   - Description: Get all users
   - Auth: Required (JWT)
   - Access: Admin only
   - Response: Array of user objects

2. GET /api/roles/users/:id
   - Description: Get user by ID
   - Auth: Required (JWT)
   - Access: Any authenticated user
   - Response: User object

3. PUT /api/roles/users/:userId/role
   - Description: Update user role
   - Auth: Required (JWT)
   - Access: Admin only
   - Body: { "role": "admin|hr|manager|employee" }
   - Response: { message: "...", user: {...} }

HOW TO USE
================================================================================

STEP 1: Restart Servers
   Backend:  cd /home/code/hrms-app && npm start
   Frontend: cd /home/code/hrms-app/client && npm start

STEP 2: Login as Admin
   - Use admin account credentials
   - Or set role via MongoDB

STEP 3: Access Role Management
   - Click "Roles" link in navbar (purple button)
   - View all users in table

STEP 4: Change Roles
   - Select new role from dropdown
   - Click "Update" button
   - See success message

STEP 5: Verify Changes
   - User's role updates immediately
   - User sees new role in navbar
   - New permissions take effect on next login

TESTING GUIDE
================================================================================

CREATE TEST ACCOUNTS:
   1. admin@test.com → Set as admin
   2. hr@test.com → Set as hr
   3. manager@test.com → Set as manager
   4. employee@test.com → Set as employee

SET ROLES VIA MONGODB:
   mongosh
   use hrms
   db.users.updateOne({email: "admin@test.com"}, {$set: {role: "admin"}})
   db.users.updateOne({email: "hr@test.com"}, {$set: {role: "hr"}})
   db.users.updateOne({email: "manager@test.com"}, {$set: {role: "manager"}})
   db.users.updateOne({email: "employee@test.com"}, {$set: {role: "employee"}})

TEST SCENARIOS:
   ✓ Login as admin → See "Roles" link
   ✓ Login as non-admin → Don't see "Roles" link
   ✓ Access /roles as admin → See all users
   ✓ Access /roles as non-admin → See error message
   ✓ Change role → See success message
   ✓ Refresh page → See updated role
   ✓ Logout/Login → See new permissions
